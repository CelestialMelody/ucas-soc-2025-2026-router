# Router v1 实现记录

本文档记录早期版本（v1）的实现方式、遇到的问题与经验教训。

---

## 一、任务需求

### 1.1 作业目标

设计并实现一个 C++ 程序，完成**2.5D 异质集成基板布线**：

- **输入**：
  - `grid_layout.json`：基板网格信息（grid_info）与所有焊点位置（top_layer / bottom_layer）
  - `netlist.json`：线网列表，每条线网包含多个焊点（bump_name）
- **输出**：
  - `output.json`：每条线网的布线段列表，每段由起点格点 `[x, y, layer]` 和终点格点组成
- **要求**：
  - 每条线网的所有焊点必须连通
  - 布线必须按网格（水平/垂直/45°对角或通孔）
  - **每个 grid 走不超过 1 根线**（硬约束）
  - 不能出现锐角（只允许 90°、135°）
  - 支持 135° 转角
  - 运行时间不超过 1 小时

### 1.2 评测方式

使用 `evaluator.py` 检查：
- `disconnected_nets`：未连通的线网
- `invalid_segments`：非法布线段
- `grid_overuse`：同一格点被多条线网占用（**这是主要问题**）
- `sharp_angles`：锐角转角
- `wirelength`：总线长
- `layers_count`：使用的层数

## 二、相关知识

### 2.1 基板布线背景

- **2.5D 异质集成**：多个芯粒（chiplet）通过基板（interposer）互连
- **网格抽象**：将基板抽象为整数坐标的网格，线只能在格点上走
- **层（Layer）**：基板有多个金属层，本作业有 Top 和 Bottom 两层（可扩展中间层）
- **通孔（Via）**：同一 (x, y) 不同层之间的连接

### 2.2 布线规则

- **同层移动**：水平、垂直、45° 对角线（`|dx|==|dy|`）
- **换层**：只能在同一 (x, y) 打通孔，不能斜着换层
- **转角**：只允许 90°、135°（不能锐角）

### 2.3 经典算法

- **Lee 算法（迷宫布线）**：BFS 从起点扩散到终点，保证找到最短路径（若存在）
- **A\* 寻路**：用启发式（如 Manhattan 距离）减少搜索范围
- **Rip-up and Reroute**：拆掉冲突线网，在更拥挤的占用图上重新布线

## 三、v1 实现方式

### 3.1 整体架构

单文件 `main.cpp`，包含：
- JSON 解析（使用 nlohmann/json）
- 数据结构：`GridPoint = (x, y, layer)`，`Segment = (start, end)`
- 占用管理：`occupancy`（已占用格点集合）、`net_to_cells`（每条线网占用的格点）
- 布线函数：`route_two_pins`、`route_two_same_layer` 等

### 3.2 核心实现步骤

#### 步骤 1：读入与解析

```cpp
// 建立 bump 名称 -> (x, y, layer) 映射
for (top_layer 中的 bump_name) → (x, y, "Top")
for (bottom_layer 中的 c4_name) → (x, y, "Bottom")
```

#### 步骤 2：线网排序

按两引脚 Manhattan 距离**升序**布线（短线网先布），减少后续冲突。

#### 步骤 3：两点布线策略（同层）

对同层两点 `(x1,y1,layer)` → `(x2,y2,layer)`，依次尝试：

1. **两种 L 形**：
   - L_hv：先水平到 `(x2, y1)`，再垂直到 `(x2, y2)`
   - L_vh：先垂直到 `(x1, y2)`，再水平到 `(x2, y2)`
   - 若某条 L 不冲突（`segment_conflicts` 检查），则使用

2. **另一层走 L**：
   - 若当前在 Top，尝试 via 到 Bottom，在 Bottom 走 L，再 via 回
   - 同样试两种 L（hv 和 vh）

3. **2 拐点绕行**（`try_detour_same_layer`）：
   - 8 个方向 × offset 1..80
   - 例如：`(x1,y1) → (x1, y2+off) → (x2, y2+off) → (x2,y2)`
   - 检查整条路径是否冲突

4. **短线网 A\***（仅对 dist ≤ 250）：
   - 在起点-终点包围盒 + margin 35 内做 A* 寻路
   - 只走未占用格点，找到则一定无 overuse

5. **重布阶段 BFS**（`g_reroute_phase = true`）：
   - 小范围 BFS（margin 75），含中间层

#### 步骤 4：多引脚线网

链式连接：`pin0 → pin1 → pin2 → ...`，每段用两点布线。

#### 步骤 5：Rip-up and Reroute

```cpp
for (iter = 0; iter < 6; ++iter) {
    收集所有参与 overuse 的线网
    for (每条冲突线网) {
        从 occupancy 中移除该线网占用的格点
        重新布线（此时其他线网已占用，需避让）
    }
}
```

### 3.3 中间层支持

- **层列表**：`kAllLayers = { "Top", "M1", "M2", "Bottom" }`（4 层）
- **A\* / BFS**：每个格点可 via 到**所有其他层**（不只是 Top↔Bottom）
- **效果**：每格点 (x, y) 有 4 个“槽位”，可被 4 条线网分别占用在不同层

### 3.4 关键函数

- `expand_segment(start, end)`：将布线段展开为经过的所有格点（用于冲突检查）
- `segment_conflicts(segs, occupancy)`：检查路径是否与已占用格点冲突
- `add_segments_to_occupancy(segs, net_name, occupancy, net_to_cells)`：将路径加入占用
- `astar_same_layer(a, b, occupancy, max_x, max_y, margin)`：A* 寻路（含中间层）
- `bounded_bfs_same_layer(...)`：BFS 寻路（重布阶段用）
- `collect_overuse_nets(net_to_cells)`：找出所有参与 overuse 的线网

## 四、遇到的问题

### 4.1 主要问题：grid_overuse_count 仍然很高

**现象**：
- C2IO1（700 条线网）上，grid_overuse_count 约 **17.3 万+**
- `summary.pass = false`，无法通过评测
- 无断路、无非法段、无锐角，但「每个 grid 超过 1 根线」的违例大量存在

**原因分析**：

1. **首轮布线策略不够“避障”**：
   - 优先用 L 形，L 形可能走“热门路径”（如某行/列），多条线网选同一 L 导致冲突
   - 只有短线网（dist ≤ 250）用 A*，长线网仍用 L/绕行，容易冲突

2. **绕行范围有限**：
   - detour 只试 offset 1..80，在非常拥挤的区域可能找不到无冲突路径
   - 绕行只在当前层，未充分利用中间层（M1/M2）做绕行

3. **Rip-up and Reroute 效果有限**：
   - 逐条拆、逐条布，拆一条时其他冲突线网仍在占用，重布时仍可能冲突
   - BFS margin 75 可能仍不够大，在拥挤区域找不到路径
   - 6 轮可能不够，但增加轮数会显著增加时间

4. **网格密度极高**：
   - C2IO1 有 700 条线网，网格 4560×3840，平均每条线网跨越数百格点
   - 即使有 4 层，在局部热点区域（如芯粒附近）仍可能 4 层都被占满

### 4.2 运行时间

- **C2IO1**：约 2 分 42 秒（4 层 + 当前策略）
- 若扩大 A* 范围或增加重布轮数，时间会显著增加（可能超过 10 分钟甚至超时）

### 4.3 其他问题

- **线长**：约 857943（网格单位），相对合理
- **层数**：4 层（Top, M1, M2, Bottom），符合“允许引入更多层数”的要求

## 五、尝试过的改进

### 5.1 已实现

1. **中间层（M1, M2）**：从 2 层 → 3 层 → 4 层，overuse 从 ~18.8 万 → ~18.2 万 → ~17.3 万
2. **短线网 A***：对 dist ≤ 250 的线网首轮用 A*，减少局部拥塞
3. **重布 BFS margin 75**：重布时搜索范围更大
4. **6 轮 Rip-up and Reroute**：多轮迭代消化冲突

### 5.2 尝试但效果有限或时间过长

1. **全量 A***：对所有线网首轮用 A*（margin 80-120）→ 超时（>10 分钟）
2. **每轮拆光所有 overuse 再逆序重布**：单轮时间过长，整体超时
3. **在其它层上做绕行**：增加大量尝试次数，时间显著增加

### 5.3 未尝试但可能有效

1. **轨道/通道分配**：先为线网分配“轨道”或“通道”，再在通道内布线
2. **全局拥塞估计**：先快速布一版，估计每格点拥塞，再优先布“拥挤区域”的线网
3. **Steiner 树优化**：多引脚线网用 Steiner 树而非链式连接
4. **分层策略**：有意将一部分线网主要布在 Top，另一部分在 Bottom/M1/M2

## 六、经验教训

### 6.1 设计原则

- **先保证合法，再优化**：必须先消除 grid_overuse（硬约束），才能谈线长、层数优化
- **时间与质量权衡**：A* / BFS 范围越大、重布轮数越多，overuse 可能越低，但时间会显著增加
- **中间层有效但有限**：增加层数能减少 overuse，但在极高密度下仍可能不够

### 6.2 算法选择

- **Lee / A\* 是基础**：网格上避障寻路的经典方法，必须实现
- **Rip-up and Reroute 是标准做法**：但需要配合好的“拆线策略”和“重布顺序”
- **简单启发式（L 形、绕行）**：速度快但容易冲突，适合“快速首轮”或“备选方案”

### 6.3 待改进方向

1. **更智能的线网顺序**：按“冲突潜力”而非仅“距离”排序
2. **全局拥塞感知**：先估计拥塞，再优先处理拥挤区域
3. **更好的重布策略**：每轮拆哪些、按什么顺序重布，需要更精细的策略
4. **多线程**：作业允许最多 4 线程，可用于并行处理不同线网或不同区域

---

## 七、总结

v1 版本实现了基本的避障布线（A*、BFS、rip-up and reroute）和中间层支持，在 C2IO1 上将 grid_overuse_count 从约 18.8 万（2 层）降到约 17.3 万（4 层），但仍**未达到 grid_overuse_count = 0** 的目标。

**核心挑战**：在极高密度的网格上（700 条线网、4560×3840 网格），仅靠“局部避障 + 多轮重布 + 中间层”难以彻底消除所有冲突，需要更全局的优化策略（如轨道分配、拥塞感知、更好的重布顺序等）。

**下一步方向**：考虑实现更全局的布线策略，或接受“用更多层数/更长线长换无 overuse”的权衡，在时间允许下进一步调参（更大 A* 范围、更多重布轮数、更多中间层）。
