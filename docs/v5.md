# 变更记录

## 当前版本 (基于 FZU 优化重构)

基于 [FZU](https://openatom.tech/opendacs2025_competition_4/bade92f99153e0a93f7dda6bfb98e2b5) 实现的核心架构（PagedGrid + PathFinder），并融合了 router.v4 中有价值的创新点。

### 与 router.v4的对比

#### 旧实现的架构

| 模块 | 实现方式 |
|------|---------|
| 网格存储 | 全尺寸平面数组 `std::vector<int/double>` |
| A* 状态 | 小网格用预分配数组 + visit_counter；大网格 (>500K) 用 `std::unordered_map` |
| 冲突解决 | PathFinder (50轮) + Cleanup (100轮) 两阶段 |
| 多引脚排序 | MST (Prim) 排序 |
| Net 排序 | 前半按 span 排序，后半随机 |
| net_type | 未解析，无层惩罚 |
| 编译优化 | 无 -O2 |
| 网格尺寸 | ceiling division: `(max + gl - 1) / gl` |

#### 当前实现的架构

| 模块 | 实现方式 |
|------|---------|
| 网格存储 | **PagedGrid** — 64×64 块按需分配，稀疏存储 |
| A* 状态 | **PagedGrid\<NodeState\>** — 统一的稀疏 O(1) 访问 |
| 冲突解决 | PathFinder + **轻量级 Cleanup**（仅扫描已布线单元） |
| 多引脚排序 | **MST (Prim) 排序**（从旧实现保留） |
| Net 排序 | 首轮顺序，后续随机 shuffle (与 FZU 一致) |
| net_type | 已支持 — type=1 的 net 在 M1/M2 层额外 +5.0 惩罚 |
| 编译优化 | **-O2 Release 模式** |
| 网格尺寸 | floor division: `max / gl` (与 FZU 一致) |

### 核心变更详情

#### 1. PagedGrid 替换平面数组（性能关键）

**问题**：旧实现为 `occupancy`、`history_cost`、`pin_net_id` 和 `AStarCache` 分配完整的平面数组。对于 C4M4 (33000×36600, grid_length=5 → 6600×7320×4层=193M 单元格)：
- occupancy: 773MB
- history_cost: 1.55GB
- pin_net_id: 773MB
- AStarCache: 3.87GB
- **总计 ~7GB**，绝大部分从未被访问

**解决**：PagedGrid 以 64×64 的块为单位按需分配，只有被访问的区域才占用内存。实际内存使用通常不到原来的 1%。

#### 2. A* 搜索状态改用 PagedGrid

**问题**：旧实现对大网格使用 `std::unordered_map<long long, double>` 存储 A* 的 `best_g` / `parent` / `visited`。unordered_map 每次访问需要哈希计算，内存碎片化严重，缓存命中率极低。

**解决**：使用 `static PagedGrid<NodeState>` 存储 A* 状态，每次搜索前调用 `freeAll()` 快速释放。O(1) 访问，良好的缓存局部性。

#### 3. Cleanup 阶段重新设计

**问题**：旧实现的 Cleanup 通过遍历整个 occupancy 数组 (`for (i=0; i<193M; ++i)`) 来查找过载单元格。这是 O(网格大小) 而非 O(布线长度)。

**解决**：新的轻量级 Cleanup 仅扫描已布线 net 的 `occupied_nodes`（O(总布线长度)），通过 `cell_to_nets` 反向索引快速定位冲突 net，使用高代价因子 (h=50, p=50) 引导路径避开拥塞区域。

#### 4. MST 多引脚排序（从旧实现保留）

对 3+ 引脚的 net，使用 Prim 最小生成树算法确定连接顺序。FZU 原版按输入顺序连接 `bumps[0]→bumps[1]→...`，这对多引脚 net 不一定最优。MST 排序通过最小化相邻引脚的曼哈顿距离来减少总线长。

#### 5. net_type 支持（FZU 特性，旧实现缺失）

解析 netlist 中的 `net_type` 字段。type=1 的 net 在 M1 (layer 0) 和 M2 (layer 1) 层额外增加 5.0 的代价惩罚，引导其使用更高层金属。

#### 6. 编译优化

CMakeLists.txt 默认使用 Release 模式 (`-O2`)，相比无优化约有 5-6 倍的速度提升。

#### 7. 输出格式改进

输出模块处理 pin 原始坐标到网格坐标的转换，支持 L 形连接段，计算并报告 wirelength、grid_overuse、sharp_angles 等指标。

### 旧实现中未采用的特性

| 特性 | 原因 |
|------|------|
| 自适应搜索边界 | 可能导致路由失败（路径需要绕行时找不到解）；PagedGrid 使全网格搜索的内存开销可控 |
| Span 排序 net 顺序 | 实测中随机 shuffle 的收敛速度已足够好，span 排序没有显著改善 |
| AStarCache + visit_counter | PagedGrid 的 freeAll() 更简洁高效，不需要复杂的缓存复用机制 |

### Benchmark 结果

| 测试用例 | 旧实现 (router.v4) | 当前实现 | FZU 参考 |
|---------|-------------------|---------|---------|
| C2IO1 (700 nets) | ~17 iter + cleanup, 数分钟 | **10 iter, 5.8s** | ~10 iter |
| C8IO1 (3300 nets) | 50 iter + 17 cleanup, 数小时 | **12 iter, 2m32s** | 12 iter |
| C4M4 (9636 nets) | 超过24小时未完成 | **13 iter, 数分钟内** | 13 iter |

### 算法参数

| 参数 | 值 |
|------|-----|
| 最大 PathFinder 迭代 | 50 |
| 初始 history_factor | 0.5 (+0.5/iter) |
| 初始 present_factor | 0.5 (×1.5/iter) |
| A* 启发式权重 | 1.2 |
| 水平/垂直移动代价 | 1.0 |
| 过孔 (层切换) 代价 | 5.0 |
| net_type=1 层惩罚 | +5.0 (M1/M2) |
| Cleanup history/present | 50.0 / 50.0 |
| Cleanup 最大迭代 | 100 |
| PagedGrid 块大小 | 64×64 |
| 金属层数 | 4 (M1, M2, M3, M4) |
